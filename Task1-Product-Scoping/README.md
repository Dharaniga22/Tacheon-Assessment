# Task 1: Product Scoping – Marketing Performance Hub



## Overview



This task focuses on scoping a solution for a common challenge faced by marketing teams: understanding marketing performance across multiple channels quickly and consistently.



The proposed solution is \*\*Marketing Performance Hub\*\*, a lightweight internal dashboard that consolidates marketing data from existing platforms and presents it in a single, easy-to-understand view.



## Why I Chose This Approach



While analyzing the problem statement, I noticed that the main issue was not a lack of data. The problem was that data already existed across multiple platforms and analysts had to manually gather and combine it before generating insights.



Because the assessment explicitly stated that existing tools and workflows should not be changed, I decided not to propose a new marketing platform. Instead, I designed a reporting layer that sits on top of the current tools and simplifies the reporting process.



## Key Decisions



### Internal Users First



I selected marketing analysts as the primary users because they currently spend the most time collecting and preparing performance reports.



Clients were considered secondary users because supporting external users would introduce additional complexity such as authentication, permissions, and support requirements.



### Dashboard-Based Solution



A dashboard was chosen because it provides a visual and centralized view of marketing performance.



This approach allows both technical and non-technical users to understand performance trends quickly without working directly with raw data.



### Focus on a Minimum Viable Product



The first version focuses only on solving the core problem:



* Consolidating marketing data
* Displaying key KPIs
* Comparing channel performance
* Providing simple performance insights



More advanced features such as AI-generated recommendations and forecasting were intentionally excluded from Version 1.



## Scope Summary



### Included



* KPI overview
* Channel comparison
* Trend monitoring
* Basic automated insights
* Data transparency
* Report export functionality



## Excluded



* AI chatbots
* Predictive forecasting
* Budget optimization
* Client self-service portal
* Mobile application



## What I Would Revisit with More Time



If additional time were available, I would:



* Create detailed wireframes
* Validate requirements with actual marketing users
* Design role-specific dashboards
* Explore AI-generated summaries
* Evaluate forecasting capabilities
